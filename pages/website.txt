Professional Practice assignments tend to prescribe an unspecific medium, and this assignment was no exception. Amongst the suggested formats were a webpage, so I decided it would be fun to create a website for the project not using WordPress or Squarespace, but from scratch by writing my own HTML, CSS and JavaScript.
Truthfully, this was not done in one session, but rather over the course of the semester. It’s a fairly simple layout with some intricate CSS, and it uses some clever JavaScript tricks to save me some time. The list of activities is written in JSON, with all pages coded to construct themselves from the list. And descriptions are saved as TXT files to be read in on page load.
Whilst this website didn’t require overwhelming innovation, it provided me with an opportunity to practice my technical skills in front-end design and try some things I hadn’t done before (namely the text read-in). I believe it’s very valuable maintaining & refining one’s skills like these. In my case it is particularly useful, as I am continuously working on my own digital portfolio (at jaimatthews.com) which features on my resume, LinkedIn, and can be the first impression for potential employers of my technical abilities.
